### Step 2: Add the Workflow File
1. In your new repository, click "Add file" ‚Üí "Create new file"

   - (Yes, include the dots and slashes - GitHub will create folders automatically)

5. Scroll down and click "Commit changes"
6. Click "Commit changes" again in the popup

### Step 3: Run the Workflow (DETAILED EXPLANATION)

**Part A - Go to Actions:**
1. At the very top of your repository, you see tabs: "Code", "Issues", "Pull requests", "Actions"
2. Click the **"Actions"** tab

**Part B - Enable Workflows (if needed):**
- **IF** you see a message "Workflows aren't being run on this repository" with a big green button:
  - Click the green button that says **"I understand my workflows, go ahead and enable them"**
- **IF** you don't see that message:
  - That's okay! Actions are already enabled, just continue below

**Part C - Run the Workflow:**
1. Look at the **LEFT side** - you'll see "All workflows" with a list
2. Click **"Generate 3D Contribution Calendar"**
3. Now look at the **RIGHT side** of the screen
4. You'll see a gray button: **"Run workflow"** ‚Üê Click this!
5. A small box/dropdown appears below it
6. Inside that box, click the green **"Run workflow"** button
7. Wait 30-60 seconds
8. Press **F5** on your keyboard to refresh
9. You should see:
   - Orange circle üü† = running
   - Green checkmark ‚úÖ = done!

- Still stuck? ‚Üí Refresh the page (F5) and try again
- Wrong repository? ‚Üí Check the URL - should be github.com/Sreeshan7/Sreeshan7

### Step 4: Fix the Workflow Error (IMPORTANT!)

**You'll see a RED X - that's normal! Here's how to fix it:**

1. At the top, you see "Workflow run was successfully requested" ‚úÖ - Good!
2. Below, you see a **RED circle with X** next to your workflow - This needs fixing!

**How to fix:**

**Option A - Update the workflow file (EASIEST):**
1. Go back to your repository main page (click "Code" tab)
2. Click on the `.github` folder
3. Click on `workflows` folder
4. Click on `profile-3d.yml` file
5. Click the pencil icon ‚úèÔ∏è (top right) to edit
6. **Delete everything** in the file
7. Copy the UPDATED content from `sreeshan-workflow.yml` again (I just fixed it!)
8. Paste it in
9. Click "Commit changes"
10. Go back to Actions tab ‚Üí Click "Run workflow" again
11. Wait 1-2 minutes ‚Üí Refresh ‚Üí Should now have GREEN checkmark ‚úÖ

**Option B - Enable workflow permissions:**
1. Click "Settings" tab (at the very top)
2. On the left menu, scroll down to "Actions" ‚Üí Click "General"
3. Scroll to bottom: "Workflow permissions"
4. Select **"Read and write permissions"**
5. Click "Save"
6. Go back to Actions ‚Üí Run the workflow again

### Step 5: Your 3D Calendar is Ready!
The workflow created a folder called `profile-3d-contrib` with your 3D calendar SVG files.
